script标签中defer和async的区别

发表于 2019-03-02

在《JavaScript高级程序设计》中给出的定义：

defer: 可选。表示脚本可以延迟到文档完全被解析和显示之后再执行。只对外部文件有效。IE7及更早版本对 嵌入脚本也支持这个属性。

async: 可选。表示应该立即下载脚本，但不应妨碍页面中的其他操作，比如下载其他资源或等待加载其他脚 本。只对外部脚本文件有效。

首先这两个属性的共同点是，告诉浏览器立即下载，但是延迟执行<script>中所指的外部文件。

两者的区别分别是：

1、defer是在 html4.01 为<script>标签定义的属性；

​ async是在 html5 为<script>标签定义的属性。

2、 如果在<script>标签中设置了defer属性，相当于告诉浏览器立即下载（并不影响其他资源的加载），但延迟执行，并且按照设置的先后顺序执行；

​ 如果在<script>标签中设置了async属性，相当于告诉浏览器立即下载（并不影响其他资源的加载），下载完后立即执行，但是不保证按照他们制定的先后顺序执行。

所以，如果设置为async属性，那么两个脚本之间最好不要有互相依赖的关系，因为所需要的依赖有可能不能在前面加载好，导致错误。
