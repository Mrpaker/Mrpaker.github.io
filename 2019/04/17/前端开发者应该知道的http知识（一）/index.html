前端开发者应该知道的http知识（一）

发表于 2019-04-17

HTTP——超文本传输协议。它用于在分布式系统之间进行通信的无状态应用层协议，是现代Web的基础。作为Web开发人员，我们都应该对此协议有深刻的理解。

让我们从Web开发人员的角度来复习这个强大的协议。我们将分成两部分来讨论这个话题。在第一部分中，我们将介绍基础知识并概述各种请求和响应头。在后面一部分，将会讲到HTTP的几个重要部分——缓存、连接处理和身份验证。

HTTP基础知识

HTTP允许各种主机和客户端之间的通信，并支持混合的网络配置。

这使得HTTP成为无状态协议。通信通常通过TCP / IP进行，但可以使用任何可靠的传输。TCP / IP的默认端口为80，但也可以使用其他端口。

主机和客户端之间的通信通过请求/响应对发生。客户端发起HTTP请求消息，作为回应通过HTTP响应消息进行服务。我们将在下一节中讨论这个基本的消息对。

该协议的当前版本是HTTP / 1.1，它为之前的1.0版本增加了一些额外的功能。在我看来，其中最重要的包括持久连接，分块传输编码和细粒度缓存头。我们将在本文中简要介绍这些功能; 第二部分将提供深入的报道。

请求方法

URL显示我们要与之通信的特定主机的标识，但应通过HTTP谓词指定应在主机上执行的操作。当然，客户希望主机执行多项操作。HTTP已经在一些上形式化，捕获了普遍适用于各种应用程序的基本要素。

这些请求动词是：

  • GET：获取现有资源。URL包含服务器查找和返回资源所需的所有必要信息。
  • POST：创建一个新资源。POST请求通常携带有效负载，指定新资源的数据。
  • PUT：更新现有资源。有效载荷可以包含资源的更新数据。
  • 删除：删除现有资源。

以上四个是最常用的请求方法，大多数工具和框架都明确地公开了这些请求方法。PUT并且DELETE有时被认为是POST方法的特殊版本，并且它们可以打包为POST具有包含确切操作的有效负载的请求：创建，更新或删除。

HTTP还支持一些较少使用的方法：

  • HEAD：这与GET类似，但没有消息体。它用于检索特定资源的服务器头，通常是通过时间戳检查资源是否已更改。
  • TRACE：用于检索请求从服务器往返的跳数。每个中间代理或网关都会将其IP或DNS名称注入Via头字段。这可用于诊断目的。
  • 选项：用于检索服务器功能。在客户端，它可以用于根据服务器可以支持的内容修改请求。

状态码

使用URL和请求方法，客户端可以向服务器发起请求。作为响应，服务器响应状态代码和消息有效负载。状态代码很重要，它告诉客户端如何解释服务器响应。HTTP规范为特定类型的响应定义了某些数字范围：

1xx：信息性消息

这类代码是在HTTP / 1.1中引入的，纯粹是临时的。服务器可以发送Expect: 100-continue消息，告诉客户端继续发送请求的剩余部分，或者忽略它是否已经发送了它。HTTP / 1.0客户端应该忽略此标头。

2xx：成功

这告诉客户端请求已成功处理。最常见的代码是200 OK。对于GET请求，服务器在消息正文中发送资源。还有其他不太常用的代码：

  • 202 Accepted：请求已被接受但可能不包括响应中的资源。这对于服务器端的异步处理很有用。服务器可以选择发送用于监视的信息。
  • 204 No Content：响应中没有消息正文。
  • 205 Reset Content：指示客户端重置其文档视图。
  • 206 Partial Content：表示响应仅包含部分内容。其他标头指示确切的范围和内容到期信息。

3xx：重定向

这要求客户采取其他措施。最常见的用例是跳转到不同的URL以获取资源。

  • 301 Moved Permanently:：资源现在位于新URL。
  • 303 See Other：资源临时位于新URL。该Location响应报头包含临时URL。
  • 304 Not Modified：服务器已确定资源未更改，客户端应使用其缓存副本。这依赖于客户端发送ETag（Enttity Tag）信息的事实，该信息是内容的散列。服务器将其与其自己的计算机进行比较ETag以检查修改。

4xx：客户端错误

当服务器认为客户端出错时，可以通过请求无效资源或发出错误请求来使用这些代码。这个类中最受欢迎的代码是404 Not Found，我认为每个人都会认同它。404表示资源无效且服务器上不存在。此类中的其他代码包括：

  • 400 Bad Request：请求格式错误。
  • 401 Unauthorized：请求需要身份验证。客户端可以使用Authorization标头重复请求。如果客户端已包含Authorization标头，则凭据错误。
  • 403 Forbidden：服务器拒绝访问资源。
  • 405 Method Not Allowed：请求行中使用的HTTP谓词无效，或者服务器不支持该动词。
  • 409 Conflict：服务器无法完成请求，因为客户端正在尝试修改比客户端时间戳更新的资源。在资源的协作编辑期间，冲突主要出现在PUT请求中。

5xx：服务器错误

此类代码用于指示处理请求时服务器故障。最常用的错误代码是500内部服务器错误。本课程中的其他人是：

  • 501 Not Implemented：服务器尚不支持所请求的功能。
  • 503 Service Unavailable：如果服务器上的内部系统出现故障或服务器过载，则可能发生这种情况。通常，服务器甚至不响应，请求将超时。
